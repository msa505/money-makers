# Stirling-PDF 变现分析

## 一句话定位
Stirling-PDF 是开源版 "iLovePDF / Adobe PDF 在线工具箱"，提供 50+ 种 PDF 处理能力（合并、拆分、压缩、OCR、签名、转换等），可自托管、可桌面、可浏览器。

## 商业模式 / 收费方式

| 收费点 | 说明 |
|--------|------|
| **SaaS 订阅** | 部署到公网，按功能/按页数/按月收费（如：免费 10 页/天，会员 ¥19/月 无限） |
| **API 调用** | 提供 PDF 处理 API，按调用量收费（适合企业集成、RPA 场景） |
| **本地增强版** | 个人用户卖桌面端 Pro 解锁更多功能或批量处理 |
| **垂直行业工具** | 合同/发票/简历 PDF 自动化处理，按企业私有化部署收费 |
| **广告/导流** | 免费版放广告，或为自建工具站导流（如 PDF 转 Word 工具站） |

## 核心功能
- 50+ PDF 工具：合并、拆分、压缩、格式转换、OCR、签名、水印、加密、页面管理
- 桌面端 + 浏览器 + 自托管服务器三端
- 支持 API 和自动化工作流
- 开源许可（GNU GPLv3），二次开发需注意许可证传染性

## 变现路径（老王可走的路）

1. **最快：PDF 工具站 + 广告/会员**
   - 部署到服务器，做 `msa505.com/pdf` 或独立域名
   - 免费使用基础功能，限制文件大小/次数；付费会员解锁大文件、批量、高级功能
   - 对标：iLovePDF、Smallpdf、PDF24

2. **API 服务：卖给开发者和企业**
   - 封装成 REST API，按调用次数/页数收费
   - 适合卖给 RPA、ERP、文档管理系统的开发者
   - 部署：Docker + Nginx + 计费网关（Stripe/支付宝）

3. **垂直场景：合同/发票处理 SaaS**
   - 用 Stirling-PDF 做 PDF 处理引擎，叠加 OCR + 模板识别
   - 卖给律所、财务、HR（如批量合同归档、发票查重）

4. **卖部署/运维服务**
   - 帮企业私有化部署 Stirling-PDF
   - 加定制功能（如 SSO、审计日志、企业品牌）

## 复刻建议

| 步骤 | 动作 |
|------|------|
| 1. 跑起来 | `docker run -d -p 8080:8080 stirlingtools/stirling-pdf` |
| 2. 套壳 | 做一个中文落地页，集成核心 5-10 个工具（别放全 50 个，聚焦转化） |
| 3. 收费 | 接入会员系统（如 Clerk/Auth.js + 支付） |
| 4. 差异化 | 加中文 OCR 支持、模板库、批量处理、企业 SSO |

## 技术栈
- 后端：Java + Spring Boot + Gradle
- 前端：HTML/JS + Thymeleaf（传统模板渲染）
- 容器：Docker 官方镜像
- 本地依赖：LibreOffice、Tesseract OCR、qpdf 等

## 风险 / 注意点
- **许可证是 GPLv3**：修改后必须开源。如果做 SaaS 收费，建议用官方 Docker 镜像不修改源码，或者只改前端/配置，避免传染性
- **内存占用较大**：Java 后端 + OCR 模型，小服务器会卡，建议 4G+ 内存
- **依赖多**：LibreOffice 等外部工具部署复杂，Docker 版最简单
- **竞争红海**：PDF 工具市场成熟，必须做垂直场景或差异化 UI 才能收钱

## 一句话总结
**PDF 工具是变现最成熟的开源赛道之一，Stirling-PDF 是现成的发动机，适合快速搭一个中文 PDF 工具站或企业 API 服务。**
